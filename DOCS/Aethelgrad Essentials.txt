🪟 Windows GDK File Locations (Important Discovery)
Minecraft moved from UWP to GDK in 2025 (around 1.21.120). File locations changed completely.
WhatOld UWP PathNew GDK PathDev packs / skins%localappdata%\Packages\Microsoft.MinecraftUWP_8wekyb3d8bbwe\LocalState\games\com.mojang\%appdata%\Minecraft Bedrock\users\shared\games\com.mojang\WorldsSame UWP path above%appdata%\Minecraft Bedrock\users\<numeric_id>\games\com.mojang\minecraftWorlds\
Key gotcha: Worlds are now per Xbox account (stored under a numeric user ID folder), not shared. The shared folder only contains packs, skins, and templates — NOT worlds. This is why minecraftWorlds appeared empty in the shared folder.

🔁 Dev Loop
What we learned about /reload
/reload does not exist on BDS. Unlike Java Edition, BDS requires a full server restart to reload scripts. The watch script handles this automatically.
Dev Workflow
1. Run watch.ps1 once (stays alive all day)
   → powershell -ExecutionPolicy Bypass -File watch.ps1

2. Edit any file in Windsurf and save

3. watch.ps1 detects the save (800ms debounce)
   → Syncs BP folder to BDS dev packs
   → Kills BDS process
   → Restarts BDS automatically

4. Script errors appear in the terminal instantly
   (filtered — only script/error lines shown, no chunk spam)

5. Connect from Minecraft client: 127.0.0.1:19132
   (stay connected across restarts — BDS reloads fast enough)
Total iteration time: ~3-5 seconds vs 30-60 seconds quit/rejoin cycle previously.

🔧 watch.ps1 — What It Does
Located at project root. Key behaviours:

Debounce: Waits 800ms after last file save before restarting — prevents 10 restarts when saving multiple files at once
Cooldown: 5 second minimum between restarts — prevents thrashing
Filtered output: Only shows lines matching Script|error|WARN|Pack Stack|Server started — suppresses chunk loading spam
Crash recovery: If BDS exits unexpectedly, waits 3s and auto-restarts
Cleanup: On Ctrl+C, kills BDS cleanly and disposes the file watcher

🟡 Design Notes & Decisions
Beta vs Stable Strategy

Local dev: "beta" (same as shipment for now — enables all features needed)
Shipment: "beta" — survives rolling BDS updates without version mismatch errors
If pinning later: Switch local to "2.6.0" for TypeScript autocomplete accuracy

Database — DynamicProperty (not entities)
Old MCE used invisible entity inventories as a database — a performance crime. Aethelgrad Essentials uses world.setDynamicProperty() / player.setDynamicProperty() exclusively. The DatabaseManager adds:

In-memory cache (reads are instant, no repeated property lookups)
Write debouncing (dirty key tracking — only writes changed keys)
Sharding for values >30KB (splits across multiple properties)
Per-player transaction queue (Promise chain prevents race conditions)

No eval / No script_eval
MCE had an eval backdoor gated only by a username check — any player named Paoeni on any server had full JS execution. Removed entirely. No script_eval capability needed or present.
No Prototype Pollution
MCE monkey-patched mc.Player.prototype and mc.World.prototype directly. Aethelgrad Essentials uses no prototype modification — all utilities are explicit imports.
